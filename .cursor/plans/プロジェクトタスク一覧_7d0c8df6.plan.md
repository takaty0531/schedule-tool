# ForClass タスク一覧（優先度順）

## 前提

- 仕様の進捗はフェーズ1〜3が完了、フェーズ4〜6が未着手
- 技術スタックは React + Vite + TypeScript + Supabase
- 参照仕様: [SPEC.md](/Users/Takato/Desktop/システム開発/家庭教師アプリ/SPEC.md)

## 優先度A（最優先: MVP拡張）

- フェーズ4のスケジュール調整機能を実装
  - 週表示UI（`week_key` / `day_index` / `slot_start`）
  - 空き時間登録・更新API連携（`slots`）
  - ルーム単位での可視化とロール別操作制御
- スケジュール調整画面ルートの追加と導線整備
  - `dashboard` → `room` → 調整画面への遷移
  - モバイル操作最適化（30分単位タップ入力）
- フェーズ4の受け入れ基準作成
  - 正常系（登録・更新・表示）
  - 異常系（権限外アクセス・不正データ）

## 優先度B（次点: 授業運用の中核）

- フェーズ5の授業記録機能を実装
  - `room/:id/records` 一覧
  - `room/:id/lesson/:lid` 詳細（`lesson_records`）
  - 宿題入力（`homework`）
- 授業ファイルアップロード実装
  - Supabase Storage `lessons` バケット連携
  - `lesson_files` のメタデータ保存
  - ファイル種別（`homework` / `material`）反映
- 画面の未実装プレースホルダ解消
  - 既存ルートに実データ表示を接続

## 優先度C（通知と自動化）

- フェーズ6 LINE通知基盤の実装
  - Edge Functions（通知送信）
  - `notification_settings` の設定UI/API
  - `pg_cron` で朝通知・授業前通知ジョブ登録
- 通知失敗時のリトライ/ログ方針を定義
  - 最低限の監視指標（成功率、失敗件数）

## 優先度D（品質・運用）

- テスト導入
  - 主要画面のコンポーネント/結合テスト方針確立
  - 認証・招待・権限制御の回帰チェック
- ドキュメント更新
  - `SPEC.md` のフェーズ進捗更新
  - 初期セットアップ・デプロイ手順の最新化
- CI品質ゲートの強化
  - `npm run lint` + `npm run build` をPR必須化

## 実装順の推奨

1. フェーズ4（調整UI + `slots`）
2. フェーズ5（授業記録 + ファイル）
3. フェーズ6（通知）
4. テスト/CI/ドキュメント整備
