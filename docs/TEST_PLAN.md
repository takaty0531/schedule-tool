# ForClass テスト方針

## 目的
- 認証、招待、権限制御、授業運用の主要フローで回帰を防ぐ
- UI変更時に `lint` / `build` に加えて、手動確認観点を固定化する

## 優先テスト観点
- 認証
  - ログイン・新規登録・パスワード再設定が期待どおり遷移する
  - 未認証ユーザーが保護ルートへ入れない
- 招待
  - 招待リンク発行、受け入れ、メンバー登録が正しく完了する
  - 無効/期限切れトークンの扱いが適切
- スケジュール
  - 30分単位で空き時間を保存できる
  - 講師が重複候補から授業を確定できる
- 授業記録・ファイル
  - 授業詳細で記録保存/更新ができる
  - ファイルのアップロード/閲覧/削除ができる
- 通知
  - 通知テンプレート保存、手動送信、授業完了通知が動作する

## 最低限の回帰チェック（PRごと）
1. `npm run lint`
2. `npm run build`
3. 変更箇所に応じた手動確認（上記優先テスト観点）
